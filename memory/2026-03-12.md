## MiniMax Status Update - RESOLVED ✅

**Root cause identified:** MiniMax has TWO different API key types:

### **1. Coding Plan Key** ❌
- **Use case:** IDE tools like Cursor only
- **Does NOT work for:** Automated bots, background scripts, Jarvis on VPS
- **Error when used incorrectly:** `code plan not support model` (error 2061)

### **2. Pay-As-You-Go General API Key** ✅
- **Use case:** Bots, scripts, automation, Jarvis
- **Where to get:** platform.minimax.io
- **Requirements:** Must have funded balance
- **Works with:** M2.5, M2.1, and other models

### **Current OpenClaw Config:**
```
Primary model: moonshot/kimi-k2.5
Fallback model: minimax/MiniMax-M2.5
Heartbeat model: moonshot/kimi-k2.5 (was Gemini, now fixed)
```

### **If MiniMax Fails:**
1. Check error code 2061 = Coding Plan key being used
2. Switch to pay-as-you-go key in `/docker/openclaw-wyqu/data/.openclaw/openclaw.json`
3. If BOTH MiniMax and Kimi fail = check credit balances:
   - platform.minimax.io (MiniMax balance)
   - platform.moonshot.cn (Kimi balance)
